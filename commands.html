⚙️ Configuration Commands
📢 !loggerchannel
Config Admin Only

Sets the channel where all ticket logs will be sent. Run this once during setup and the bot will automatically deliver every log to that channel.

!loggerchannel <channel_id>
Parameters
<channel_id>The ID of the Discord channel to receive logs. Right-click any channel → "Copy Channel ID" (requires Developer Mode).
👥 !loggerrole
Config Admin Only

Designates one or more roles whose members can trigger ticket logging. When a user with any of these roles interacts with a ticket, it gets logged automatically. You can add as many roles as you need — there's no limit.

!loggerrole @Role1 [@Role2 @Role3 ...]
Parameters
@Role1The first role to assign. Mention it directly with @ in your message.
@Role2+Optional. Add more roles separated by spaces. No maximum role count.

💡 Quick Tips

  • →Enable Developer Mode in Discord settings to copy channel and user IDs.
  • →You can re-run !loggerchannel or !loggerrole at any time to update your settings.
  • →The bot needs Send Messages permission in your log channel to work properly.
  • →Multiple roles work great for servers with different staff tiers (Support, Mod, Admin, etc).
